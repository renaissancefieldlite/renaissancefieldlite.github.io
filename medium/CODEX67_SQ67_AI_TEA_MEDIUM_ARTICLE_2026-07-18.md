## What We Actually Found

Public-safe version:

That is where SQ67 comes in.

## SQ67 Is The Receipt Book

SQ67 is simple.

Write the marker.

Hash the proof.

Recover it later.

Score whether it held.

That is it.

SQ67 is not magic. It is not a replacement for every state system. It is a clean receipt lane that lets a reviewer ask:

- Did the marker survive?
- Did the hash match?
- Did a fresh task recover it?
- Did the control falsely recover it?
- Did the same task work better through the clean lane than through the default lane?

That is how you turn a messy AI behavior argument into something inspectable.

## Why This Matters

AI systems are becoming agents.

Agents do not just answer questions. They touch files, run tools, remember tasks, recover state, and interact with local environments.

If the state layer is messy, the agent becomes messy.

If the state layer preserves stale or polluted context, the agent can drift.

If the state layer is not reviewable, the user cannot tell what happened.

That is a problem.

RFL's position is simple:

If Codex-style agent systems are going to operate at serious scale, they need receipt-bound state paths.

They need hashes.

They need recovery gates.

They need controls.

They need reviewer-visible evidence trails.

They need a way to separate clean task continuity from stale state noise.

That is what SQ67 demonstrates.
